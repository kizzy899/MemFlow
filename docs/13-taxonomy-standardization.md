# 标签体系标准化

MemFlow 在入库前规范 AI 分类，在 API、Notion 和导出边界再次规范历史值。一级分类固定为 AI、编程开发、英语学习、金融财务、论文写作、工具效率、项目灵感、生活经验、职业发展、其他；二级分类、内容类型、来源平台和重要程度见 `app/taxonomy.py`。

别名采用不区分大小写的精确/包含匹配，URL 平台判断优先于 AI 值；未知一级/二级分别降级为“其他”/“未分类”。关键词支持列表或分隔字符串，去空、大小写去重、单项 20 字、最多 8 个。新增分类需同时修改词表、别名、测试和 Notion Select 选项。

持久化继续使用既有字段和英文枚举；未新增迁移。失败规范化采用上述安全默认值，不抛外部 API 错误。测试覆盖别名、未知值、平台、重要程度、关键词及整组结果。
## Agent 面试标签

二级分类新增 `Agent面试`。当分类值或内容上下文同时包含 Agent/智能体与面试、求职、岗位或上岸语义时，规范为 `Agent面试`，并将同名关键词加入 Notion 的“关键词”multi-select。普通 Agent 技术内容仍归入 `Agent开发`。

## 文本内链接识别

`input_type=text` 会提取正文首个 HTTP(S) URL，剔除句末中英文标点，保存到既有 `source_url`/`normalized_url` 字段，并按域名识别来源平台。`xhslink.com` 与 `xiaohongshu.com` 均识别为小红书。重复文本的历史条目若尚无链接，会回填链接、将 Notion 状态转为 pending 并尝试更新已有页面；链接唯一性冲突时不覆盖另一条记录。

该增强未新增数据库字段或迁移。无链接时仍保持“手动输入”；非法 URL 不进入提取结果后的规范化流程。测试覆盖链接清理、小红书识别、Agent 面试分类和关键词标签。
